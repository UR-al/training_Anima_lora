General args

Model files — DiT / text encoder / VAE

Point at your model files (e.g. forge-neo / ComfyUI) so the repo needs no local models/. Blank = the bundled default (models/…). Used for both training and auto-preprocess caching.

More general args — precision · grad checkpoint/accum · batch · dataloader · speed / VRAM (attn_mode, blocks_to_swap, vae_chunk, unsloth…)

Network args

LyCORIS args — shown per algo

Anima DiT: use networks.lycoris_anima + an anima-* preset to wrap the transformer blocks (the stock presets target diffusers names the Anima DiT doesn't use). conv_*/use_tucker are hidden — Anima has no conv layers.

REPA alignment — vision-encoder feature alignment (any LoRA variant)

Turning this on auto-caches PE-Spatial features at preprocess ({stem}_anima_pe_spatial.safetensors). Works with networks.lora_anima and lycoris_anima.

More network args — network/rank/module dropout · train-on (unet/te) · min/max timestep (t_min/t_max) · scale_weight_norms · base/merge weights

Bucket args

Preprocessing for the subsets defined in Subset args below. Anima uses constant-token buckets — kohya's min/max bucket reso & steps don't apply.

Resolution tiers (target_res)

Anima uses constant-token buckets — pick which tier edges to allow.

Multi-scale: each image is resized+cached into every checked tier (or only tiers it's large enough for, if "Skip upscaling" is on), one [[datasets]] block per tier per subset. Needs ≥2 tiers checked.

Masking (masked loss — ignore text / focus subject)

More bucket / caching args — target_res · resize_interpolation · use_vae_cache / use_text_cache · sample_ratio · path_pattern

Subset args

Subsets — image folder(s) + per-subset settings

Entering / picking a dataset .toml auto-fills the subsets & tiers. To import a LoRA_Easy / kohya config, use "Import config file" in Queue & saved configs below.

More subset args — caption_dropout / caption_tag_dropout · reg_data_dir · caption_separator (per-subset repeat / keep_tokens / flip / random_crop / pad are on each subset card above)

Noise offset args

Anima is flow-matching (rectified flow) — classic noise_offset / pyramid-noise don't apply. The equivalent input-perturbation knob is ip_noise_gamma (~0.1).

Sample args

Per-prompt resolution / seed / negative / steps / CFG / guidance / shift are tokens inside the sample_prompts file. Use the editor below — or set the path manually. Samplers: euler / er_sde (per-prompt --ss in Extra, or the global sample_sampler arg).

Train Anima args

Anima-DiT specifics — flow-matching (timestep_sampling / discrete_flow_shift / sigmoid_*) + tokenizer. T5 tokenizer: leave blank to use the bundled one. Model paths (DiT / Qwen3 / VAE) and speed/VRAM knobs (attn_mode, vae_chunk, blocks_to_swap, unsloth) live in General args.
